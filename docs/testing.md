# Тестирование

Проект использует xUnit для backend-тестов.

## Назначение тестов

Тесты проверяют application-логику без запуска HTTP-сервера и без подключения к реальным PostgreSQL/MongoDB. Такой подход позволяет быстро проверять игровые расчёты, сценарии персонажа и доступ к комнатам.

## Запуск

```bash
dotnet test dnd-helper.sln
```

## Расположение

```text
Backend/dnd-helper.Tests
├── Application
└── Features
```

## Покрытые области

### Dice

`DiceRoller` проверяется на:
- корректный разбор dice expressions;
- flat values;
- положительные и отрицательные modifiers;
- invalid expressions;
- ограничения безопасности.

### Character rest

`CharacterRestService` проверяется на:
- короткий отдых;
- длительный отдых;
- восстановление hit points;
- восстановление spell slots;
- обработку неподдерживаемого типа отдыха.

### Character spells

`CharacterSpellService` проверяется на:
- применение известного персонажу заклинания;
- расход ячейки заклинаний;
- применение cantrip без расхода ячейки;
- запрет применения заклинания, которого персонаж не знает;
- запрет применения leveled spell при исчерпанных ячейках.

### Room access

`RoomAccessService` проверяется на:
- доступ участника комнаты;
- доступ владельца;
- отказ пользователю без прав;
- обработку ролей.

## Детерминированные броски

Для тестирования бросков используется deterministic roller:

```csharp
var roller = new DiceRoller((minimumInclusive, maximumExclusive) => 4);
```

Такой подход делает тесты стабильными и позволяет проверять итоговые значения лечения, урона и бросков.

## Принципы тестирования

- Тестировать application-сервисы напрямую.
- Использовать fake/stub для внешних зависимостей.
- Не требовать реальных баз данных для unit-level проверок.
- Делать dice rolls детерминированными.
- Проверять успешные сценарии и ошибки валидации.
